# Pirates Expedition Mini App - Component Library

Welcome to the **Pirates Expedition Mini App** component library! This Storybook showcases all the reusable components used throughout the application, following our modern architecture patterns.

## Architecture Overview

This application follows a **Container/Presenter pattern** with **focused hook composition** and a **layered service architecture**. Components are organized into three main categories:

### 1. UI Components (`components/ui/`)
Reusable, primitive UI elements that can be used anywhere in the application:
- **PirateButton** - Styled buttons with multiple variants (primary, secondary, danger)
- **PirateCard** - Card containers with theming support
- **DeadlineTimer** - Countdown timer with urgency indicators

### 2. Feature Components
Components specific to features, organized by domain:

#### Dashboard Components (`components/dashboard/`)
- **DashboardStats** - Statistics overview cards
- **DashboardPresenter** - Main dashboard layout and state rendering
- **ExpeditionTimeline** - Timeline view of expeditions

#### Expedition Components (`components/expedition/`)
- **ExpeditionCard** - Individual expedition card with progress
- **ExpeditionProgress** - Progress bar with analytics
- **ItemsGrid** - Grid display of expedition items
- **ConsumeItemModal** - Modal for consuming items
- **Wizard Components** - Multi-step expedition creation

### 3. Container Components (`containers/`)
Logic-only components that compose hooks and delegate rendering to presenters:
- **DashboardContainer** - Dashboard data fetching and orchestration
- **CreateExpeditionContainer** - Expedition creation workflow
- **ExpeditionDetailsContainer** - Single expedition details

## Component Patterns

### Container/Presenter Separation

**Containers** handle:
- Hook composition and orchestration
- Data fetching and state management
- Action handler delegation
- **NO UI rendering** (except delegation to Presenter)

**Presenters** handle:
- Pure UI rendering based on props
- Conditional rendering (loading, error, success states)
- Layout composition
- **NO business logic or data fetching**

Example:
```typescript
// Container - Logic only
export const DashboardContainer: React.FC = () => {
  const { expeditions, loading, error } = useExpeditions();
  const stats = useDashboardStats(expeditions);
  const actions = useDashboardActions();

  return (
    <DashboardPresenter
      loading={loading}
      error={error}
      stats={stats}
      expeditions={expeditions}
      actions={actions}
    />
  );
};

// Presenter - UI only
export const DashboardPresenter: React.FC<Props> = ({
  loading,
  error,
  stats,
  expeditions,
  actions
}) => {
  if (loading) return <LoadingScreen />;
  if (error) return <ErrorScreen error={error} />;

  return (
    <Layout>
      <DashboardStats stats={stats} />
      <ExpeditionTimeline expeditions={expeditions} />
    </Layout>
  );
};
```

## Design System

### Colors
The application uses a **pirate-themed color palette**:
- **Primary**: `#e94560` (Pirate Red)
- **Secondary**: `#0f3460` (Deep Ocean Blue)
- **Background**: `#1a1a2e` (Dark Navy)
- **Surface**: `#16213e` (Darker Blue)
- **Text**: `#f1f1f1` (Light Gray)

### Typography
- **Headings**: Pirate-themed font family
- **Body**: System font stack for readability

### Spacing
Consistent spacing scale based on `0.25rem` increments:
- `xs`: 0.25rem
- `sm`: 0.5rem
- `md`: 1rem
- `lg`: 1.5rem
- `xl`: 2rem
- `2xl`: 3rem
- `3xl`: 4rem
